Requirements

The server must have these programs/settings to run Fusion Ticket:

  • PHP version =>5.2.1 through 5.3.1, tested with 5.3.0.  We do NOT support: PHP <5.2.1 or >5.3.1!
  • MySQL version >5.0 with MySQLi is Mandatory. Tested with version 5.1.35
  • Enable email support in PHP - Check email security settings.
  • Adobe Acrobat Reader (to view/print PDF files)
  • cURL should be enabled for some features (PayPal, Authorize.net)
  • openssl should be enabled for some features i.e. https:
  • XML parser should be enabled
  • iconv support for non-latin text in PDF templates

Some third party code is already included with the distribution:

  • Smarty (see smarty.php.net) is installed in .../smarty folder
  • Html2Pdf in .../Html2pdf
  • HTMLMimeMail in .../classes
  • Fusion Ticket provides all required pear components in .../includes/pear folder.

Fusion Ticket has been tested on CentOS, Fedora Core, Ubuntu, and Windows with Apache 2.0. Generally, the latest stable version of software will have no problems but there are no guarantees.

If there are any problems with the above requirements, or the web host, contact us here.

Fusion Ticket can provide hosting which is always up-to-date for security and compatibility. Use the same version that we design the software on. All we ask is that you donate regularly to the project.

We do not recommend the Beta Versions of Fusion Ticket for Production Use!

Full Installation

  • Download the archive Fusion Ticket and decompress it somewhere in the web root path of the web server:
  • This produces a directory called Fusionticket - you can rename it.
  • Create new database using the MySQL client, for example: mysqladmin create Fusionticket
  • Set the following folders and subfolders to writable 

        .../includes/config/*
        .../includes/fonts/*
        .../includes/temp/*
        .../files/*
  • Browse to the installation script /yourhost/Fusionticket/inst/index.php and select ‘Full Installation’
  • Reset the .../includes/config folder to Read-only.

Upgrade to Beta6

  • Make a backup of your current database, it will not be backwards compatible.
  • Download the archive Fusion Ticket Beta6
  • Assuming the name of the Beta5 folder is /Fusionticket/ rename it to ../Fusionticket_B5/ (or whatever you like).
  • Create a new folder and give it the name of the original folder, in this case ../Fusionticket/
  • Copy the downloaded files to the new /Fusionticket/ folder.
  • Copy the the original /files folder to the new folder
  • Check the new .tpl files against any you may have previously modified.
    Copy the changed files to the custom folder.
  • Copy the original file init_config.php, now located in ../Fusionticket_B5/includes/config/ to the new folder.
    Remove the $_SHOP->root and $_SHOP->root_secure fields those are now automatically filled.
  • Starting with Beta6 it will expect you to use a secure (SSL) site for handling the checkout pages.
    Enable this by add $_SHOP->secure_site= 1; in init_config.php (contect your ISP for more information.)
    People will see a warning when the checkout-page is not secure.
  • Set the following folders and subfolders to writable 
    .../includes/config/init_config.php
.../includes/fonts/*
.../includes/temp/*
.../files/*
  • Browse to the installation script ../yourhost/Fusionticket/inst/index.php and select ‘Upgrade’
  • Reset the .../includes/config folder to Read-only.
  • Now you are ready to use beta6.
